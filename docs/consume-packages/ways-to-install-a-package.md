## <a name="what-happens-when-a-package-is-installed"></a><span data-ttu-id="6fd30-128">Co się stanie, gdy pakiet jest zainstalowany</span><span class="sxs-lookup"><span data-stu-id="6fd30-128">What happens when a package is installed</span></span>

<span data-ttu-id="6fd30-129">Był wyświetlany różne narzędzia NuGet zazwyczaj utworzyć odwołanie do pakietu w pliku projektu lub `packages.config`, następnie wykonać przywracania pakietów skuteczne pakiet zostanie zainstalowany.</span><span class="sxs-lookup"><span data-stu-id="6fd30-129">Simply said, the different NuGet tools typically create a reference to a package in the project file or `packages.config`, then perform a package restore, which effectively installs the package.</span></span> <span data-ttu-id="6fd30-130">Wyjątek stanowi `nuget install`, który rozwija tylko pakiet do `packages` folder i wszystkie inne pliki nie powoduje modyfikacji.</span><span class="sxs-lookup"><span data-stu-id="6fd30-130">The exception is `nuget install`, which only expands the package into a `packages` folder and does not modify any other files.</span></span>

<span data-ttu-id="6fd30-131">Ogólny proces jest następująca:</span><span class="sxs-lookup"><span data-stu-id="6fd30-131">The general process is as follows:</span></span>

1. <span data-ttu-id="6fd30-132">(Wszystkie narzędzia, z wyjątkiem `nuget.exe`) Zapisz identyfikator pakietu i wersję do pliku projektu lub `packages.config`.</span><span class="sxs-lookup"><span data-stu-id="6fd30-132">(All tools except `nuget.exe`) Record the package identifer and version into the project file or `packages.config`.</span></span>

2. <span data-ttu-id="6fd30-133">Pobieranie pakietu:</span><span class="sxs-lookup"><span data-stu-id="6fd30-133">Acquire the package:</span></span>
   - <span data-ttu-id="6fd30-134">Sprawdź, czy pakiet (przez dokładny identyfikator oraz numer wersji) jest już zainstalowany w *globalnymi pakietami* folderu zgodnie z opisem na [Zarządzanie globalnymi pakietami i folderami pamięci podręcznej](managing-the-global-packages-and-cache-folders.md).</span><span class="sxs-lookup"><span data-stu-id="6fd30-134">Check if the package (by exact identifer and version number) is already installed in the *global-packages* folder as described on [Managing the global packages and cache folders](managing-the-global-packages-and-cache-folders.md).</span></span>

   - <span data-ttu-id="6fd30-135">Jeśli pakiet nie znajduje się w *globalnymi pakietami* folderze próbują pobrać go ze źródeł wymienione na liście [pliki konfiguracyjne](Configuring-NuGet-Behavior.md).</span><span class="sxs-lookup"><span data-stu-id="6fd30-135">If the package is not in the *global-packages* folder, attempt to retrieve it from the sources listed listed in the [configuration files](Configuring-NuGet-Behavior.md).</span></span> <span data-ttu-id="6fd30-136">Dla źródeł online próbował najpierw pobrać pakiet z pamięci podręcznej, chyba że `-NoCache` jest określony za pomocą `nuget.exe` poleceń lub `--no-cache` jest określony za pomocą `dotnet restore`.</span><span class="sxs-lookup"><span data-stu-id="6fd30-136">For online sources, attempt first to retrieve the package from the cache unless `-NoCache` is specified with `nuget.exe` commands or `--no-cache` is specified with `dotnet restore`.</span></span> <span data-ttu-id="6fd30-137">(Visual Studio i `dotnet add package` zawsze używać pamięci podręcznej.) Jeśli pakiet jest używany z pamięci podręcznej, "Pamięć PODRĘCZNĄ" pojawia się w danych wyjściowych.</span><span class="sxs-lookup"><span data-stu-id="6fd30-137">(Visual Studio and `dotnet add package` always use the cache.) If a package is used from the cache, "CACHE" appears in the output.</span></span> <span data-ttu-id="6fd30-138">Pamięć podręczna zawiera 30-minutowy czas wygaśnięcia.</span><span class="sxs-lookup"><span data-stu-id="6fd30-138">The cache has an expiration time of 30 minutes.</span></span>

   - <span data-ttu-id="6fd30-139">Jeśli pakiet nie jest w pamięci podręcznej, próba go pobrać ze źródła wymienione w konfiguracji.</span><span class="sxs-lookup"><span data-stu-id="6fd30-139">If the package is not in the cache, attempt to download it from the sources listed in the configuration.</span></span> <span data-ttu-id="6fd30-140">Jeśli pakiet został pobrany, "GET" i "OK" są wyświetlane w danych wyjściowych.</span><span class="sxs-lookup"><span data-stu-id="6fd30-140">If a package is downloaded, "GET" and "OK" appear in the output.</span></span>

   - <span data-ttu-id="6fd30-141">Jeśli pakiet nie może pomyślnie uzyskana z dowolnymi źródłami, instalacja nie powiedzie się na tym etapie ze względu na błąd taki jak [NU1103](../reference/errors-and-warnings/NU1103.md).</span><span class="sxs-lookup"><span data-stu-id="6fd30-141">If the package cannot be successfully acquired from any sources, installation fails at this point with an error such as [NU1103](../reference/errors-and-warnings/NU1103.md).</span></span> <span data-ttu-id="6fd30-142">Uwaga tego błędy z `nuget.exe` Pokaż polecenia tylko ostatni źródło opcja jest zaznaczona, ale oznacza, że pakiet nie była dostępna z dowolnego źródła.</span><span class="sxs-lookup"><span data-stu-id="6fd30-142">Note that errors from `nuget.exe` commands show only the last source checked, but implies that the package wasn't available from any source.</span></span>

   <span data-ttu-id="6fd30-143">Podczas uzyskiwania pakietu, mogą mieć zastosowanie kolejność źródeł w konfiguracji NuGet:</span><span class="sxs-lookup"><span data-stu-id="6fd30-143">When acquiring the package, the order of sources in the NuGet configuration may apply:</span></span>

   - <span data-ttu-id="6fd30-144">W przypadku projektów przy użyciu formatu PackageReference NuGet sprawdza źródła lokalnego folderu i udziałów sieciowych przed sprawdzeniem źródła HTTP.</span><span class="sxs-lookup"><span data-stu-id="6fd30-144">For projects using the PackageReference format, NuGet checks sources local folder and network shares before checking HTTP sources.</span></span>

   - <span data-ttu-id="6fd30-145">Dla projektów przy użyciu `packages.config` zarządzania formatu NuGet używa kolejności źródeł w konfiguracji.</span><span class="sxs-lookup"><span data-stu-id="6fd30-145">For projects using the `packages.config` management format, NuGet uses the order of the sources in the configuration.</span></span> <span data-ttu-id="6fd30-146">Wyjątek stanowi operacje przywracania, w którym to przypadku porządkowania źródła jest ignorowany i NuGet korzysta z pakietu z dowolnego źródła odpowiada w pierwszej kolejności.</span><span class="sxs-lookup"><span data-stu-id="6fd30-146">An exception is restore operations, in which case source ordering is ignored and NuGet uses the package from whichever source responds first.</span></span>

   - <span data-ttu-id="6fd30-147">Ogólnie rzecz biorąc kolejność, w którym NuGet kontroli źródła nie jest szczególnie istotne, ponieważ dowolnego danego pakietu przy użyciu określonego identyfikatora oraz numer wersji jest dokładnie taka sama, niezależnie od źródła został znaleziony.</span><span class="sxs-lookup"><span data-stu-id="6fd30-147">In general, the order in which NuGet checks sources isn't particularly meaningful, because any given package with a specific identifier and version number is exactly the same on whatever source it's found.</span></span>

3. <span data-ttu-id="6fd30-148">(Wszystkie narzędzia, z wyjątkiem `nuget.exe`) Zapisz kopię pakietu oraz inne informacje w *pamięci podręcznej http* folderu zgodnie z opisem na [Zarządzanie globalnymi pakietami i folderami pamięci podręcznej](managing-the-global-packages-and-cache-folders.md).</span><span class="sxs-lookup"><span data-stu-id="6fd30-148">(All tools except `nuget.exe`) Save a copy of the package and other information in the *http-cache* folder as described on [Managing the global packages and cache folders](managing-the-global-packages-and-cache-folders.md).</span></span>

4. <span data-ttu-id="6fd30-149">Jeśli pobrane, należy zainstalować pakiet w poszczególnych użytkowników *globalnymi pakietami* folderu.</span><span class="sxs-lookup"><span data-stu-id="6fd30-149">If downloaded, install the package into the per-user *global-packages* folder.</span></span> <span data-ttu-id="6fd30-150">NuGet tworzy osobny podfolder dla każdego identyfikatora pakietu, a następnie tworzy oddzielne podfoldery dla każdej zainstalowanej wersji pakietu.</span><span class="sxs-lookup"><span data-stu-id="6fd30-150">NuGet creates a subfolder for each package identifier, then creates subfolders for each installed version of the package.</span></span>

5. <span data-ttu-id="6fd30-151">Aktualizacja innych projektów, plików i folderów:</span><span class="sxs-lookup"><span data-stu-id="6fd30-151">Update other project files and folders:</span></span>

    - <span data-ttu-id="6fd30-152">W przypadku projektów przy użyciu funkcji PackageReference, zaktualizuj wykres zależności pakietu, które są przechowywane w `obj/project.assets.json`.</span><span class="sxs-lookup"><span data-stu-id="6fd30-152">For projects using PackageReference, update the package dependency graph stored in `obj/project.assets.json`.</span></span> <span data-ttu-id="6fd30-153">Zawartość pakietu, sami nie są kopiowane do dowolnego folderu projektu.</span><span class="sxs-lookup"><span data-stu-id="6fd30-153">Package contents themselves are not copied into any project folder.</span></span>
    - <span data-ttu-id="6fd30-154">Dla projektów przy użyciu `packages.config`, skopiuj te części pakietu rozwinięty, które odpowiadają platforma docelowa projektu do projektu `packages` folderu.</span><span class="sxs-lookup"><span data-stu-id="6fd30-154">For projects using `packages.config`, copy those parts of the expanded package that match the project's target framework into project's `packages` folder.</span></span> <span data-ttu-id="6fd30-155">(Korzystając z `nuget install`, cały pakiet rozszerzonej jest kopiowana, ponieważ `nuget.exe` nie analizuje pliki projektu, aby zidentyfikować platformę docelową.)</span><span class="sxs-lookup"><span data-stu-id="6fd30-155">(When using `nuget install`, the entire expanded package is copied because `nuget.exe` does not examine project files to identify the target framework.)</span></span>
    - <span data-ttu-id="6fd30-156">Aktualizacja `app.config` i/lub `web.config` Jeśli korzysta z pakietu [źródła i konfiguracji pliku przekształcenia](../create-packages/source-and-config-file-transformations.md).</span><span class="sxs-lookup"><span data-stu-id="6fd30-156">Update `app.config` and/or `web.config` if the package uses [source and config file transformations](../create-packages/source-and-config-file-transformations.md).</span></span>

6. <span data-ttu-id="6fd30-157">Jeśli zainstalować wszystkie zależności niskiego poziomu nie już istnieje w projekcie.</span><span class="sxs-lookup"><span data-stu-id="6fd30-157">Install any down-level dependencies if not already present in the project.</span></span> <span data-ttu-id="6fd30-158">Ten proces może aktualizować wersje pakietów w procesie, zgodnie z opisem w [rozpoznawania zależności](../consume-packages/dependency-resolution.md).</span><span class="sxs-lookup"><span data-stu-id="6fd30-158">This process might update package versions in the process, as described in [Dependency Resolution](../consume-packages/dependency-resolution.md).</span></span>

7. <span data-ttu-id="6fd30-159">(Tylko visual Studio) Wyświetl plik readme pakietu, jeśli to możliwe, w oknie programu Visual Studio.</span><span class="sxs-lookup"><span data-stu-id="6fd30-159">(Visual Studio only) Display the package's readme file, if available, in a Visual Studio window.</span></span>
